> 이제는 해야 한다… 바이브 코딩.
> 사실 이미 하고 있긴 하지만,
> **‘배워본 적은 없는 상태’**에 가까웠던 것 같다.

그동안 나는 바이브 코딩 도구 중에서도
웹 기반의 챗봇을 주로 사용해 왔다.
무료이기도 하고 접근성이 좋았던 **ChatGPT**가 대표적이었다.

솔직히 말하면
> *“ChatGPT로 궁금한 거 물어보고 답변 받는 정도면
나한테는 충분하지 않을까?”*

라고 생각하고 있었다.

그런데 AI의 발전은
내가 생각한 것보다 훨씬 빠르고, 훨씬 입체적이었다.

주변을 보면 이미
바이브 코딩을 통해 **실제 결과물을 만들고**,
문제를 해결하고,
작은 서비스 단위까지 구현해내는 사람들이 늘어나고 있었다.

그걸 보면서 자연스럽게 들었던 생각은 하나였다.

> **“아, 이건 그냥 써보는 수준을 넘어서
제대로 한 번 파봐야겠다.”**

그래서 오늘,
**바이브 코딩 도전기 1일차**를 시작했다.

---

## AI Assistant란?

- 프로그래밍 과정에서
질문에 답하고
코드를 함께 작성해 주는 **가상 개발 파트너**

단순히 “답변해 주는 도구”라기보다는,
**함께 생각하고 같이 만들어가는 존재**에 가깝다.

---

## 바이브 코딩이란?

- 사람에게 설명하듯 자연어로 요구사항을 전달하면
- AI가 코드를 제시하고
- 사용자가 이를 검토·수정하며 완성해 나가는 개발 방식

키보드보다 **대화가 중심이 되는 개발 경험**이라고 느껴졌다.

---

## 바이브 코딩 도구들

### 웹 기반 챗봇
- 설치 없이 웹에서 바로 사용 가능
- 결과물을 복사 → 붙여넣기 해야 하는 구조
- 접근성이 가장 좋음

**ex)** Claude, ChatGPT, Gemini

---

### 모델 선택형
- 여러 언어 모델을 선택해 사용할 수 있음
- 강력하지만 비용이 이중으로 발생할 수 있음

**ex)** Cursor, Replit, Windsurf

---

### 전용 모델형
- 특정 언어 모델을 로컬에 설치해 사용하는 방식
- 로컬 파일 직접 접근 가능
- 개발자 친화적이지만 진입장벽 존재

**ex)** Claude Code, Gemini CLI

---

## 오늘 사용한 도구: Claude

앞으로는 **Claude**를 중심으로
바이브 코딩을 진행해 보려고 한다.

Claude에는 `아티팩트(Artifact)`라는
꽤 인상적인 기능이 있는데,
이게 단순히 대화만 하는 수준을 넘어서
**실제로 작동하는 결과물을 바로 확인**할 수 있게 해준다.

## 2. `프롬프트` 입력하기

프롬프트는 단순한 지시문이 아니라,
> *“내가 어떤 결과물을 원하는지
내 생각을 구체화하는 설계도”*

에 가깝다.

## 4. 커스터마이즈 & 재게시

디자인이나 기능을 다시 수정할 수 있고,
수정 후 `게시` 버튼을 누르면

## 느낀 점

바이브 코딩으로
페이지를 만들고 → 디자인 수정하고 → 배포까지 하는 데
**5분도 채 걸리지 않았다.**

이쯤 되니까 이런 생각이 들었다.

> “진짜… 나 이제 뭐 먹고 살지?”

그런데 곰곰이 생각해보면
이런 고민을 한다는 것 자체가
**그만큼 더 성장할 수 있는 지점에 와 있다는 뜻** 아닐까.

새로운 걸 배운다는 건
역시 재미있다.

이제 시작이니까,
앞으로는
- 프롬프트를 어떻게 짜야 하는지
- 어디까지 맡기고, 어디서부터 내가 개입해야 하는지
- 실제 개발에 어떻게 녹일 수 있을지

하나씩 정리해 보려고 한다.

**바이브 코딩 도전기, 이제 시작이다.**
